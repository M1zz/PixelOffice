# PixelOffice MVP 기능 우선순위 결정

- **상태**: 완료
- **생성일**: 2024년 2월 5일 오전 10:00
- **참여자**: 리이오, 마이클, 제니

## 배경

PixelOffice 1.0 출시를 위한 핵심 기능 우선순위를 정해야 합니다. 리소스가 제한되어 있어 3개의 핵심 기능만 선택해야 합니다.

## 독립 의견

### 리이오 (기획)

## 핵심 기능 우선순위 제안

### 1순위: AI 직원 대화 시스템
사용자가 AI 직원과 자연스럽게 대화할 수 있는 기능이 PixelOffice의 핵심 가치입니다.

### 2순위: 프로젝트 관리
칸반 보드와 태스크 관리는 실제 업무 효율을 보여주는 핵심입니다.

### 3순위: 부서별 전문성
각 부서(기획, 디자인, 개발, QA, 마케팅)의 전문가 역할이 차별화 포인트입니다.

### 마이클 (개발)

## 기술적 관점에서의 우선순위

### 1순위: 데이터 영속성
앱 재설치 후에도 데이터가 유지되어야 합니다. 현재 이 부분이 취약합니다.

### 2순위: Claude Code 연동
API 없이 Claude Code CLI를 활용하는 것이 기술적 차별점입니다.

### 3순위: 오프라인 지원
로컬 데이터 저장으로 네트워크 없이도 기본 기능 사용 가능해야 합니다.

### 제니 (디자인)

## 사용자 경험 관점 우선순위

### 1순위: 픽셀 캐릭터 시스템
귀여운 픽셀 캐릭터가 사용자 감성을 자극합니다. 이게 없으면 그냥 챗봇이에요.

### 2순위: 직관적인 오피스 뷰
가상 오피스 공간에서 직원들이 움직이는 모습이 몰입감을 줍니다.

### 3순위: 사원증 시스템
각 직원의 개성과 통계를 볼 수 있는 사원증이 캐릭터 애착을 높입니다.

## 교차 검토

### 리이오 (기획)

## 다른 의견에 대한 검토

마이클의 데이터 영속성 지적에 전적으로 동의합니다. 아무리 좋은 기능도 데이터가 날아가면 무용지물입니다.

제니의 픽셀 캐릭터 의견도 타당합니다. 감성적 요소가 없으면 차별화가 어렵습니다.

### 수정된 제안
1. 데이터 영속성 (기반)
2. AI 대화 + 픽셀 캐릭터 (핵심 경험)
3. 프로젝트 관리 (실용성)

## 종합 결과

### 요약

PixelOffice MVP의 3대 핵심 기능으로 (1) 데이터 영속성, (2) AI 대화 + 픽셀 캐릭터 시스템, (3) 기본 프로젝트 관리를 선정했습니다. 기술적 안정성을 기반으로 감성적 차별화와 실용적 가치를 모두 제공하는 방향입니다.

### 합의 사항

- 데이터 영속성은 모든 기능의 기반이 되어야 함
- 픽셀 캐릭터가 PixelOffice의 핵심 차별화 요소임
- AI 대화 기능은 핵심 가치 제안의 중심

### 쟁점

- 프로젝트 관리 vs 오피스 뷰 중 어느 것이 더 중요한지
- Claude Code 연동의 우선순위

### 액션 아이템

- [높음] 데이터 저장 시스템 구축: datas 폴더에 모든 데이터를 JSON/MD로 저장하여 앱 재설치에도 데이터 유지
- [보통] 픽셀 캐릭터 다양화: 더 많은 캐릭터 외형과 애니메이션 추가
- [높음] AI 대화 품질 개선: 부서별 전문가 프롬프트 고도화

### 핵심 인사이트

- 기술적 안정성과 감성적 차별화의 균형이 중요
- 사용자가 캐릭터에 애착을 느끼면 자연스럽게 재방문
- 데이터가 보존되어야 장기 사용자 확보 가능
