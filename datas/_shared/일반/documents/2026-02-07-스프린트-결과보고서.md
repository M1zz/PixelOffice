## 3. 기획팀 작업 상세 내역

### 3.1 PRD 작성 과정

**접근 방식**:
1. PROJECT.md에서 프로젝트 비전, 타겟 사용자, 핵심 기능, 성공 기준 재확인
2. 이전 대화에서 확정된 "자동 개발 파이프라인" 핵심 콘셉트 기반으로 구조화
3. 5개 Phase (입력 → 해석 → 실행 → 배포 → 피드백)로 파이프라인 단계 설계
4. 각 Phase별 기능 요구사항(FR)과 수용 기준 정의
5. MVP와 향후 확장을 명확히 분리

**핵심 설계 결정사항**:

| 결정 사항 | 선택 | 근거 |
|-----------|------|------|
| AI 엔진 | Claude Code CLI | macOS 로컬 환경에서 가장 자연스러운 연동 방식 |
| 데이터 저장 | 로컬 파일 시스템 (JSON/MD) | 1인 사용자 앱, 서버 불필요, 빠른 MVP |
| MVP 배포 범위 | 로컬 빌드까지 | App Store 배포 자동화는 리스크 높아 후순위 |
| 자동 수정 루프 | 최대 3회 | 무한 루프 방지 + 실패 시 사용자 개입 |
| 프로젝트 관리 | Git 기반 | 자연스러운 롤백, 이력 관리 |

### 3.2 MVP 기능 범위 결정 근거

**포함 기준**: "사용자가 요구사항 한 줄을 입력하면 코드가 수정되고 빌드가 성공하는" 최소 흐름

```
[MVP 핵심 흐름]
사용자 입력 → 의도 분류 → 작업 분해 → 코드 생성(Claude Code) → 빌드 확인 → 결과 표시
```

**제외 근거**:
- 이미지 입력: 텍스트만으로도 핵심 가치 검증 가능
- 자동 배포: 보안/서명 이슈로 리스크 높음, 빌드 성공까지만으로 충분
- 기획/디자인 AI 자동 실행: 개발 자동화가 먼저 검증되어야 함
